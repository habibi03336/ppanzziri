# 1. 한 문장 정의

> 이 서비스는 3,000만원으로 얼마나, 어떻게 살아가고 있는지를 보여주는 생존 계기판이다.
> 

가계부가 아니다.

금융 서비스가 아니다.

공개 생존 실험을 시각화하는 대시보다.

# 2. 디자인 목표

> 사용자가 3초 안에 현재 잔액을 이해하게 만든다.
> 

모든 디자인 결정은 이 목표에 종속된다.

# 3. 핵심 지표 우선순위

1. 현재 잔액 (가장 크게)
2. 잔액 추이(생존 곡선)
3. 하루 평균 비용
4. 태그별 금액 비중
5. 기록 목록

목록은 주인공이 아니다.

대시보드가 주인공이다.

# 4. 정보 구조

## 홈(공개 조회)

상단부터 아래 순서:

1. Hero 카드
    - 현재 잔액
    - 3,000만원 대비 남은 비율
    - 누적 수입 / 누적 지출
    - 상위 태그 요약
    - 캐릭터(홈에서만 사용)
2. 생존 곡선
    - 잔액 라인 차트
    - 3,000만원 기준선
    - 30일 / 전체 토글
3. 하루 비용
    - 전체 평균
    - 최근 30일 평균
    - 유효구간 기준 계산
4. 태그 비중
    - 상위 태그 스택바
5. 최근 기록 요약
    - 날짜별 아코디언
    - 전체 기록으로 이동
6. 잔액 인증
    - 최신순 썸네일

# 5. 시간 해석 모드

상단 토글 제공:

- 거래 기준
    - 실제 돈이 이동한 흐름
    - 잔액 변화 기준
- 유효 기준
    - 돈이 효력을 가진 시점 기준
    - 하루 평균 비용 계산 기준

잔액은 항상 거래 기준 현실을 따른다.

해석만 달라진다.

# 6. 입력 화면 방향

입력은 모바일앱 콘솔처럼 설계한다.

구성:

1. 거래 정보
    - 지출 / 수입
    - 거래일
    - 총 금액
2. 유효구간
    - 여러 개 추가 가능
    - 합계는 총 금액과 일치
    - “회차 자동 생성(균등 분할)” 기능 제공
3. 태그
    - 기존 태그 선택 가능
    - 새 태그 생성 가능
    - 합계는 총 금액과 일치
4. 인증 사진(선택)

입력은 최소 단계로 구성하되, 의미는 손실되지 않아야 한다.

# 9. 컬러 전략

> 외부는 브랜드 크림슨, 내부는 무채.
> 
> 
> 색은 감정을 만들지 않고, 변화를 설명한다.
> 

이 서비스는 강한 브랜드 프레임과 담백한 내부 UI의 대비로 구성된다.

### 브랜드 색

- Brand Crimson: `#C4002F`
- 모바일 화면의 외부 프레임 및 웹 배경에 사용
- 활성 상태, 강조 라인, 선택 상태에 제한적으로 사용

브랜드 색은 “껍질”이다.

앱 내부 전체를 붉게 만들지 않는다.

### 내부 UI 기본 색

- 앱 배경: `#F2F2F2`
- 카드: `#FFFFFF`
- 기본 텍스트: `#111111`
- 보조 텍스트: `#666666`
- 구분선: `#E5E5E5`

내부는 항상 절제된 무채 중심으로 유지한다.

### 데이터 색

- 수입 / 회복: `#1F3A8A`
- 지출 / 감소: `#C4002F`
- 잔액: 무채(검정 계열)

잔액은 감정 없이 보여준다.

레드는 긴장을, 블루는 회복을 의미한다.

### 사용 원칙

- 화면 전체를 레드로 채우지 않는다.
- 내부 배경에 브랜드 레드를 사용하지 않는다.
- 고채도, 형광, 그라데이션 사용은 지양한다.
- 색은 최소, 숫자는 명확하게.

> 붉은 프레임 안에 담백한 생존 기록이 들어 있다.
>
